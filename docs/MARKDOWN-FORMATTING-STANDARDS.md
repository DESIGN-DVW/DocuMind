# Markdown Formatting Standards

**Version:** 1.0.0
**Created:** 2026-03-10
**Status:** Active — Enforced across all DVWDesign repositories

---

## Overview

This document defines the mandatory markdown formatting rules for all DVWDesign repositories. These rules are enforced by a combination of:

- **markdownlint** (standard rules via `.markdownlint.json`)
- **DVW custom rules** (auto-fixable via `.markdownlint-cli2.jsonc`)
- **Custom fix scripts** (`fix-markdown.mjs`, `fix-custom-errors.mjs`)
- **Pre-commit hooks** (lint-staged + husky)
- **VSCode integration** (inline errors + format-on-save)

---

## Rule 1: Table Separator Spacing (DVW001)

Table separator rows **MUST** have one space between the pipe and the dashes on each side.

```text
✅ CORRECT — spaces inside each separator cell:
| Endpoint | Method | Description |
| ---------- | -------- | ------------- |

❌ WRONG — compact format, no spaces:
| Endpoint | Method | Description |
| ---------- | -------- | ------------- |
```

### Alignment markers

Alignment colons are allowed and follow the same spacing rule:

```text
| Left | Center | Right |
| :--- | :---: | ---: |
```

### Why this rule exists

- Claude Code and other AI tools tend to generate compact `|---|` separators
- The markdownlint formatter and VSCode extension expect the spaced format
- Without this rule, every table generated by an agent requires manual correction

### Enforcement

| Layer              | Tool                                       | Auto-fixes?          |
| ------------------ | ------------------------------------------ | -------------------- |
| Linter             | markdownlint + DVW001 custom rule          | Yes (`--fix`)        |
| Pre-commit         | lint-staged runs `markdownlint-cli2 --fix` | Yes                  |
| VSCode             | DavidAnson.vscode-markdownlint extension   | Yes (format-on-save) |
| Cron / daemon      | `npm run fix:custom`                       | Yes                  |
| Agent instructions | CLAUDE.md + markdown-fixer agent           | Prevention           |

### Custom rule file

[config/rules/table-separator-spacing.cjs](../config/rules/table-separator-spacing.cjs)

- Rule ID: `DVW001` / `table-separator-spacing`
- Tags: `table`
- Provides `fixInfo` for automatic correction
- Loaded via `.markdownlint-cli2.jsonc` and `.vscode/settings.json`

### Pattern definition

[config/custom-error-patterns.json](../config/custom-error-patterns.json) — pattern `table-3`

```json
{
  "id": "table-3",
  "name": "Table Separator Spacing",
  "severity": "error",
  "fixStrategy": "automated"
}
```

## Rule 2: Fenced Code Block Identifiers

**Every** fenced code block **MUST** have a language identifier. Empty ` ``` ` blocks are forbidden.

### Specific languages

When the content language is known, use the specific identifier:

`javascript`, `typescript`, `bash`, `json`, `python`, `css`, `html`, `yaml`

### Default hierarchy

When no specific language applies, use this priority order:

| Priority | Identifier | Use when |
| --- | --- | --- |
| 1 | `md` | Content is markdown (headings, lists, links, emphasis) |
| 2 | `diagram` | Content is a diagram (Mermaid, FigJam, flowcharts) |
| 3 | `text` | Last resort — plain text, terminal output, non-code content |

### Examples

```text
✅ CORRECT:

` ` `javascript
const x = 1;
` ` `

` ` `md
## Section Title
- Item one
` ` `

` ` `diagram
graph TD
    A --> B
` ` `

` ` `text
Some plain text output
` ` `

❌ WRONG:

` ` `
const x = 1;
` ` `
```

### Enforcement

| Layer              | Tool                                    | Auto-fixes?        |
| ------------------ | --------------------------------------- | ------------------ |
| Linter             | markdownlint MD040                      | No (flags only)    |
| Fix script         | `fix-markdown.mjs` (language detection) | Yes (auto-detects) |
| Agent instructions | CLAUDE.md + markdown-fixer agent        | Prevention         |

### Language detection

[scripts/fix-markdown.mjs](../scripts/fix-markdown.mjs) — `detectLanguage()` function

Detection order (first match wins):

1. `javascript` — `const`, `let`, `function`, `import`, `export`, `=>`
2. `typescript` — `interface`, `type`, `: string`, `: number`
3. `bash` — `#!/bin/bash`, `$`, `npm`, `cd`, `mkdir`
4. `json` — `{`, `[`, `"`
5. `python` — `def`, `class`, `print(`
6. `yaml` — `---`, `version:`, `name:`
7. `css` — `color:`, `display:`, `;`
8. `html` — `<div`, `<span`, `<!DOCTYPE`
9. `diagram` — `graph`, `flowchart`, `sequenceDiagram`, `classDiagram`, etc.
10. `md` — `#`, `##`, `-`, `>`, `[`, `**`
11. `text` — default fallback

## Configuration Files

### .markdownlint-cli2.jsonc (root)

Single source of truth for markdownlint-cli2. Auto-discovered by:

- CLI (`npm run lint` / `npm run lint:fix`)
- VSCode extension (DavidAnson.vscode-markdownlint)
- lint-staged (pre-commit hooks)

```jsonc
{
  "config": {
    "extends": "./config/.markdownlint.json"
  },
  "customRules": [
    "./config/rules/table-separator-spacing.cjs"
  ],
  "ignores": ["node_modules", "index", ".git", "dist", "build"]
}
```

### config/.markdownlint.json

Standard markdownlint rules. Key enforced rules:

| Rule | Purpose |
| --- | --- |
| MD001 | Heading levels increment by one |
| MD003 | ATX-style headings (`#` not underline) |
| MD022 | Blank lines around headings |
| MD031 | Blank lines around fenced code blocks |
| MD032 | Blank lines around lists |
| MD040 | Code blocks must have language identifier |
| MD047 | Files end with single newline |

### .vscode/settings.json

```jsonc
{
  "markdownlint.config": {
    "extends": "./config/.markdownlint.json"
  },
  "markdownlint.customRules": [
    "./config/rules/table-separator-spacing.cjs"
  ],
  "[markdown]": {
    "editor.defaultFormatter": "DavidAnson.vscode-markdownlint",
    "editor.formatOnSave": true
  }
}
```

Available via `Cmd+Shift+P` > `Tasks: Run Task`:

| Task                             | Command                                  | Scope           |
| -------------------------------- | ---------------------------------------- | --------------- |
| Validate Markdown                | `npm run lint`                           | All `.md` files |
| Fix Markdown                     | `npm run lint:fix`                       | All `.md` files |
| Fix Table Separators             | `npm run fix:custom`                     | All `.md` files |
| Fix All Markdown (lint + custom) | `npm run lint:fix && npm run fix:custom` | All `.md` files |


## NPM Scripts

| Script | Purpose |
| --- | --- |
| `npm run lint` | Check all markdown (standard + DVW001) |
| `npm run lint:fix` | Auto-fix all markdown (standard + DVW001) |
| `npm run fix` | Legacy fix script (line breaks, code blocks, bold/italic) |
| `npm run fix:custom` | Custom fixes (tables, headings, whitespace, horizontal rules) |
| `npm run fix:custom:all` | Custom fixes across all DVWDesign repositories |
| `npm run fix:custom:dry-run` | Preview custom fixes without writing |

### Recommended fix sequence

```bash
# 1. Standard linting + DVW001 auto-fix
npm run lint:fix

# 2. Custom pattern fixes (table separators, heading levels, etc.)
npm run fix:custom

# 3. Legacy fixes (code block languages, bold-to-heading conversion)
npm run fix
```

## Agent Integration

### Where agents read the rules

1. **CLAUDE.md** — loaded into every Claude Code session in this project
2. **`.claude/agents/markdown-fixer.md`** — CRITICAL section at top of agent definition

### Key instruction for agents

When generating markdown:

- Tables: always use `| --- |` separator format (spaces required)
- Code blocks: always specify a language (`md` > `diagram` > `text` as defaults)
- These rules apply to ALL markdown output, not just files in this repository

## Adding New Custom Rules

### 1. Create the rule file

```bash
# config/rules/my-new-rule.cjs
module.exports = {
  names: ['my-rule-name', 'DVW002'],
  description: 'What this rule checks',
  tags: ['relevant-tag'],
  parser: 'none',
  function: function rule(params, onError) {
    for (let i = 0; i < params.lines.length; i++) {
      const line = params.lines[i];
      // Detection logic...
      if (violationDetected) {
        onError({
          lineNumber: i + 1,
          detail: 'Explanation',
          fixInfo: {
            lineNumber: i + 1,
            editColumn: 1,
            deleteCount: line.length,
            insertText: fixedLine,
          },
        });
      }
    }
  },
};
```

## 2. Register the rule

Add to `.markdownlint-cli2.jsonc`:

```jsonc
"customRules": [
  "./config/rules/table-separator-spacing.cjs",
  "./config/rules/my-new-rule.cjs"
]
```

Add to `.vscode/settings.json`:

```jsonc
"markdownlint.customRules": [
  "./config/rules/table-separator-spacing.cjs",
  "./config/rules/my-new-rule.cjs"
]
```

### 3. Add pattern definition

Add entry to `config/custom-error-patterns.json` for documentation and `fix-custom-errors.mjs` integration.

### 4. Test

```bash
# Create a test file with the violation
echo '...' > /tmp/test.md

# Verify detection
npx markdownlint-cli2 /tmp/test.md

# Verify auto-fix
npx markdownlint-cli2 --fix /tmp/test.md
cat /tmp/test.md
```
