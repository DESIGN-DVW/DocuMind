<objective>
Add graceful shutdown and DB-aware health check to the DocuMind daemon, and make the file watcher polling-aware for Docker compatibility.

Purpose: Docker sends SIGTERM on `docker stop`. Without a handler, the container hangs for 10 seconds then gets SIGKILL (exit 137), leaving the SQLite WAL in an inconsistent state. The /health endpoint currently only proves the process is alive, not that the DB is reachable — the Docker HEALTHCHECK needs the DB probe.

Output: Modified daemon/server.mjs with shutdown handler and health probe; modified daemon/watcher.mjs with usePolling support.
</objective>

Modify the chokidar `watch()` options in daemon/watcher.mjs (line 57-65) to add polling support for Docker Desktop bind mounts.

Add to the options object passed to `watch()`:

```javascript

usePolling: process.env.CHOKIDAR_USEPOLLING === 'true',
interval: parseInt(process.env.CHOKIDAR_INTERVAL || '1000', 10),

```

These go alongside the existing `ignored`, `persistent`, `ignoreInitial`, and `awaitWriteFinish` options.

When `CHOKIDAR_USEPOLLING` is unset or not `'true'`, `usePolling` will be `false` — no behavior change on macOS native. When set to `'true'` in docker-compose.yml (Plan 02), chokidar will use stat polling instead of inotify, which is required for file change detection on Docker Desktop VirtioFS bind mounts.

Do NOT remove or modify the existing `awaitWriteFinish` option — it serves a different purpose (debouncing rapid writes).
