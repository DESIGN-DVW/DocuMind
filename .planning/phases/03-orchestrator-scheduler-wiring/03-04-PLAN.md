### **1. Add three detection functions to `orchestrator.mjs`:**

### **detectSimilarities(db, ctx):**

  - Use TF-IDF cosine similarity on keyword score vectors (reuse data from keywords table)

  - Strategy: For each pair of documents in the same repository, compute cosine similarity of their keyword vectors

  - Only compare docs within the same repo (cross-repo is expensive and less useful)

  - Threshold: 0.7 (only store pairs with similarity >= 0.7)

  - Clear old auto-detected similarities first: `DELETE FROM content_similarities WHERE notes LIKE '%auto_detected%'`

  - Insert matches: `INSERT INTO content_similarities (doc1_id, doc2_id, similarity_score, deviation_type, notes) VALUES (?, ?, ?, 'potential_duplicate', '{"auto_detected": true}')`

  - Implementation approach:

    ```javascript

    function detectSimilarities(db, ctx) {
      const docs = db.prepare(`
        SELECT d.id, d.repository, k.keyword, k.score
        FROM documents d JOIN keywords k ON k.document_id = d.id
      `).all();

      // Build keyword vectors per document
      const vectors = new Map(); // docId -> Map<keyword, score>
      for (const row of docs) {
        if (!vectors.has(row.id)) vectors.set(row.id, { repo: row.repository, kws: new Map() });
        vectors.get(row.id).kws.set(row.keyword, row.score);
      }

      // Cosine similarity for same-repo pairs
      const docIds = [...vectors.keys()];
      let similarCount = 0;

      db.prepare("DELETE FROM content_similarities WHERE notes LIKE '%auto_detected%'").run();

      const insert = db.prepare(`
        INSERT INTO content_similarities (doc1_id, doc2_id, similarity_score, deviation_type, notes)
        VALUES (?, ?, ?, 'potential_duplicate', '{"auto_detected": true}')
      `);

      const batch = db.transaction(() => {
        for (let i = 0; i < docIds.length; i++) {
          for (let j = i + 1; j < docIds.length; j++) {
            const a = vectors.get(docIds[i]);
            const b = vectors.get(docIds[j]);
            if (a.repo !== b.repo) continue; // same repo only

            const sim = cosineSimilarity(a.kws, b.kws);
            if (sim >= 0.7) {
              insert.run(docIds[i], docIds[j], Math.round(sim * 1000) / 1000);
              similarCount++;
            }
          }
        }
      });
      batch();

      console.log(`[orchestrator] Found ${similarCount} similar document pairs`);
      return similarCount;
    }

    ```

  - Add `cosineSimilarity(vecA, vecB)` helper that computes dot product / (|A| * |B|) over the keyword Maps.

### **detectDeviations(db, ctx):**

  - Detect 5 deviation types across all documents:

  - Clear old auto-detected deviations: `DELETE FROM deviations WHERE resolver = 'auto'`

  - **content_drift**: Docs in same folder with same heading structure but different content length (>50% variance) — suggests one is outdated

  - **structure_change**: Docs with `modified_at` in last 7 days that have changed heading count vs previous scan (compare `content` headings count)

  - **rule_violation**: Docs missing required frontmatter fields (title, date/created) — simple check

  - **version_mismatch**: Docs with `version` in frontmatter where the version string doesn't match the latest in the same folder

  - **metadata_inconsistency**: Docs where `frontmatter.category` disagrees with `classification` column

  - Insert each deviation with appropriate severity: rule_violation=medium, content_drift=low, structure_change=low, version_mismatch=medium, metadata_inconsistency=low

  - Return count of deviations found
