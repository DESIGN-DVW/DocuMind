### backfill-summaries.mjs:

Export a single function `backfillSummaries(db)` that accepts an open better-sqlite3 database instance (the same one from migrate.mjs).

Implementation:

1. Query all documents where `summary IS NULL`: `SELECT id, path, frontmatter, content FROM documents WHERE summary IS NULL`

2. For each document, extract summary using the locked extractive hierarchy:

   - **Priority 1:** Parse `frontmatter` (it's a JSON string in the DB). If parsed object has a `description` field with string value > 20 chars, use it (truncated to 200 chars).

   - **Priority 2:** Split `content` by newlines. Find the first line that is non-empty, does not start with `#`, does not start with `|`, does not start with `` ``` ``, and has length > 30 chars. Use it (truncated to 200 chars).

   - **Priority 3:** Use the filename from the path (`path.split('/').pop().replace(/\.md$/i, '')`).

3. Batch update in chunks of 500 using `db.transaction()`:

   ```javascript

   const update = db.prepare('UPDATE documents SET summary = ? WHERE id = ?');
   const batchUpdate = db.transaction((batch) => {
     for (const doc of batch) {
       update.run(doc.summary, doc.id);
     }
   });

   ```

4. Log progress with chalk: "Backfilling summaries: {N}/{total} documents..."

5. Return the count of documents updated.

Note on frontmatter parsing: The `frontmatter` column stores a JSON string (not raw YAML). Use `JSON.parse()`, not gray-matter. gray-matter was used during initial indexing to extract frontmatter — the result is already serialized as JSON in the DB. Wrap in try/catch in case of malformed JSON — fall through to Priority 2.

### backfill-classifications.mjs:

Export a single function `backfillClassifications(db)` that accepts an open better-sqlite3 database instance.

Implementation:

1. Define classification rules as an array of `{ pattern: RegExp, classification: string }`:

   ```javascript

   const CLASSIFICATION_RULES = [
     { pattern: /\/docs\/api\//,        classification: 'engineering/api-docs' },
     { pattern: /CLAUDE\.md$/,          classification: 'engineering/architecture' },
     { pattern: /\/\.planning\//,       classification: 'engineering/architecture' },
     { pattern: /ADR[-_]/i,             classification: 'engineering/architecture/adrs' },
     { pattern: /README\.md$/i,         classification: 'references/readme' },
     { pattern: /CHANGELOG/i,           classification: 'operations/changelog' },
     { pattern: /\/scripts\//,          classification: 'engineering/scripts' },
     { pattern: /\/config\//,           classification: 'engineering/config' },
     { pattern: /\/tests?\//,           classification: 'engineering/tests' },
     { pattern: /\/docs\//,             classification: 'guides/documentation' },
     { pattern: /\.github\//,           classification: 'operations/ci-cd' },
     { pattern: /package\.json$/,       classification: 'engineering/config' },
   ];

   ```

   Rules are checked in order — first match wins. No match = `'uncategorized'`.

2. Query all documents where `classification IS NULL`: `SELECT id, path FROM documents WHERE classification IS NULL`

3. For each document, match path against rules. Assign classification.

4. Batch update in chunks of 500 using `db.transaction()`.

5. Log progress and return count updated.

6. After updates, log a summary of classification distribution:

   ```javascript

   const dist = db.prepare('SELECT classification, COUNT(*) as cnt FROM documents GROUP BY classification ORDER BY cnt DESC').all();

   ```

   Print the top 10 categories with counts.

Update `scripts/db/migrate.mjs` to call backfill scripts after all SQL migrations complete.

Add the following logic AFTER the migration file loop completes but BEFORE `db.close()`:

```javascript

// Run backfill scripts if any migrations were applied this run
if (appliedCount > 0) {
  console.log(chalk.blue('\nRunning post-migration backfill...'));

  const { backfillSummaries } = await import('./backfill/backfill-summaries.mjs');
  const summaryCount = backfillSummaries(db);
  console.log(chalk.green(`  Backfilled ${summaryCount} document summaries`));

  const { backfillClassifications } = await import('./backfill/backfill-classifications.mjs');
  const classCount = backfillClassifications(db);
  console.log(chalk.green(`  Backfilled ${classCount} document classifications`));

  // FTS5 rebuild after bulk writes (mandatory — research Pitfall 1)
  console.log(chalk.blue('  Rebuilding FTS5 index...'));
  db.exec("INSERT INTO documents_fts(documents_fts) VALUES('rebuild')");
  console.log(chalk.green('  FTS5 index rebuilt'));
}

```

Track `appliedCount` — increment it each time a migration is applied in the loop. The backfill only runs when new migrations were applied (not on idempotent re-runs where everything is skipped).

The backfill scripts are idempotent themselves (`WHERE summary IS NULL` / `WHERE classification IS NULL`), so running migrate.mjs again after a partial backfill will only process remaining NULL rows.

After wiring, run the full migration:

```bash

node scripts/db/migrate.mjs

```

This should:
