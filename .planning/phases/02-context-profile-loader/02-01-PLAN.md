```javascript
/**
 * Compute the longest common directory prefix from an array of absolute paths.
 * Returns '' if paths is empty.
 *
 * Example: commonDir(['/a/b/c/d', '/a/b/c/e', '/a/b/f']) => '/a/b'
 */
export function commonDir(paths) {
  if (!paths.length) return '';
  const parts = paths[0].split('/');
  let common = '';
  for (let i = 0; i < parts.length; i++) {
    const candidate = parts.slice(0, i + 1).join('/');
    if (paths.every(p => p.startsWith(candidate + '/'))) common = candidate;
    else break;
  }
  return common;
}
```

This utility is consumed by both `daemon/server.mjs` and `daemon/watcher.mjs` in Plan 02 to derive REPOS_ROOT from `ctx.repoRoots`. It MUST be defined here (not duplicated in each consumer).

Create `context/loader.mjs` that exports a single async function `loadProfile(profilePath?)`:

1. **Resolve file path**: `profilePath` argument > `process.env.DOCUMIND_PROFILE` > `path.resolve(import.meta.dirname, '../config/profiles/dvwdesign.json')` (use `import.meta.dirname` which is available in Node 20+, or fall back to `fileURLToPath(import.meta.url)` dirname pattern if needed).

2. **Read file**: `await fs.readFile(filePath, 'utf-8')`. On ENOENT/EACCES, throw a descriptive Error:
   ```text
   [DocuMind] Cannot load context profile from "{filePath}": {err.message}
   Set DOCUMIND_PROFILE env var to a valid profile JSON path.
   ```

3. **Parse JSON**: `JSON.parse(raw)`. On SyntaxError, throw:
   ```text
   [DocuMind] Profile at "{filePath}" is not valid JSON: {err.message}
   ```

4. **Validate with Zod**: `profileSchema.parse(json)`. Let ZodError propagate naturally — the caller (server.mjs) will catch it and call process.exit(1). Do NOT catch ZodError in the loader.

5. **Build ctx object** via a `buildCtx(validated, filePath)` async helper:
   - If `validated.repositories` exists, map to `repoRoots: [{ name, path }]` filtering by `active !== false`
   - If `validated.repositoryRegistryPath` exists, resolve the path relative to the profile file's directory (use `path.resolve(path.dirname(filePath), validated.repositoryRegistryPath)`), read it, parse it, and build `repoRoots` from `registry.repositories.filter(r => r.active !== false).map(r => ({ name: r.name, path: path.join(registry.basePath, r.path) }))`. If the registry file cannot be read, throw an error mentioning the resolved path.
   - Compile `classificationRules`: map each `{ pattern, classification }` to `{ pattern: new RegExp(rule.pattern), classification: rule.classification }`
   - Copy `keywordTaxonomy`, `relationshipTypes`, `lintRules` (with default `{ profile: 'standard' }` if lintRules is undefined)
   - Set `profileId: validated.id`

6. **Freeze and return**: `Object.freeze(ctx)`. Shallow freeze is sufficient per research.

The module must have NO side effects at import time — no Sets created, no files read, no globals mutated. Everything happens inside `loadProfile()`.

Export only `loadProfile`.
