<task type="auto">
  <name>Task 2: Create GitHub Actions publish workflow</name>
  <files>.github/workflows/publish.yml</files>
  <action>
Create `.github/workflows/publish.yml` — the tag-triggered multi-arch build and GHCR push workflow.

The workflow must:

1. Trigger on `push: tags: ['v*.*.*']` only (NOT on branch pushes)

2. Use `ubuntu-latest` runner with `timeout-minutes: 60` (ARM64 QEMU compilation is slow)

3. Set `permissions: contents: read, packages: write` on the job

4. Define env vars: `REGISTRY: ghcr.io` and `IMAGE_NAME: ${{ github.repository }}`

Steps (in order):

1. `actions/checkout@v5`

2. `docker/setup-qemu-action@v4` — enables ARM64 emulation

3. `docker/setup-buildx-action@v5` — activates buildx builder

4. `docker/login-action@v4` — authenticates to GHCR with `registry: ghcr.io`, `username: ${{ github.actor }}`, `password: ${{ secrets.GITHUB_TOKEN }}`

5. `docker/metadata-action@v6` (id: meta) — extracts tags and OCI labels:

   - `images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}`

   - `flavor: latest=auto`

   - `tags: type=semver,pattern={{version}}` and `type=semver,pattern={{major}}.{{minor}}`

6. `docker/build-push-action@v7` — builds and pushes:

   - `context: .`

   - `platforms: linux/amd64,linux/arm64`

   - `push: true`

   - `tags: ${{ steps.meta.outputs.tags }}`

   - `labels: ${{ steps.meta.outputs.labels }}`

   - `cache-from: type=gha`

   - `cache-to: type=gha,mode=max`

Add a comment at the top of the file referencing the Docker multi-platform CI docs. Name the workflow "Publish Docker Image" and the job "build-and-push".
  </action>

  </verify>
