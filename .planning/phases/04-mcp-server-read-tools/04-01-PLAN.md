<task type="auto">
  <name>Task 2: Create daemon/mcp-server.mjs with all 6 read tools and update PM2 config</name>
  <files>daemon/mcp-server.mjs, ecosystem.config.cjs</files>
  <action>
    Create `daemon/mcp-server.mjs` as follows:

    **Line 1-2 (CRITICAL — before any imports):**
    Redirect console.log to stderr. This MUST be the very first executable code:
    ```javascript
    // Redirect stdout -> stderr so JSON-RPC wire is never polluted
    console.log = (...args) => process.stderr.write(args.join(' ') + '\n');
    ```

    **Imports:**
    ```javascript
    import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
    import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
    import Database from 'better-sqlite3';
    import path from 'path';
    import { fileURLToPath } from 'url';
    import { z } from 'zod';
    import { loadProfile } from '../context/loader.mjs';
    import { findRelated } from '../graph/relations.mjs';
    ```

    **DB setup:**
    Open with `readonly: true`. Do NOT set WAL pragma on readonly connection (it's a no-op per research open question #2). Set `foreign_keys = ON`.
    ```javascript
    const __dirname = path.dirname(fileURLToPath(import.meta.url));
    const ROOT = path.resolve(__dirname, '..');
    const DB_PATH = process.env.DOCUMIND_DB || path.join(ROOT, 'data/documind.db');
    const db = new Database(DB_PATH, { readonly: true });
    db.pragma('foreign_keys = ON');
    ```

    **Profile loading:**
    ```javascript
    const ctx = await loadProfile();
    ```

    **Server creation:**
    ```javascript
    const server = new McpServer({ name: 'DocuMind', version: '3.0.0' });
    ```

    **Register 6 tools (all follow the same pattern — try/catch with structured error returns):**

    1. **search_docs** (MCPR-02): Full-text search. Adapt SQL from server.mjs /search endpoint. Add `classification` filter (LIKE prefix match). Use `snippet(documents_fts, 3, '[', ']', '...', 32)` for MCP (use brackets not HTML marks). Add `d.classification` to SELECT. Input: query(string), repo(string?), category(string?), classification(string?), limit(int 1-100, default 20). Description: "Full-text search across all DocuMind-indexed documentation. Returns ranked results with path, repository, classification, and content snippet. Use this to find existing docs before creating new ones."

    2. **get_related** (MCPR-03): Direct wrapper around `findRelated(db, docId, maxDepth)`. Input: doc_id(int), hops(int 1-3, default 2). Enforce hops max 3 in Zod schema. Add `LIMIT 200` safety: after calling findRelated, slice results to 200 entries. Description: "Get documents related to a given document ID (from search_docs results) via relationship graph traversal. Returns paths, relationship types, and traversal depth up to N hops."

    3. **get_keywords** (MCPR-04): Adapt SQL from server.mjs /keywords endpoint. Input: repo(string?), category(string? — "technology", "action", or "topic"), limit(int 1-200, default 50). Description: "Get TF-IDF keyword cloud. Filter by repository and/or keyword category (technology, action, topic). Returns keywords ranked by score."

    4. **get_tree** (MCPR-05): Adapt SQL from server.mjs /tree endpoint. Input: repo(string, required). Description: "Get folder hierarchy for a repository. Returns classified folder structure with document counts and folder types."

    5. **check_existing** (MCPR-06): New logic — search FTS5 for query, compute a normalized score from FTS5 rank. The FTS5 rank column is a negative float (closer to 0 = more relevant). Normalize: `score = Math.max(0, Math.min(1, 1 - Math.abs(rank) / 20))` (dividing by 20 as baseline — tunable later). Return `{ query, exists: boolean, confidence: number, matches: [...] }` where `exists = topScore >= threshold`. Input: query(string), repo(string?), threshold(number 0-1, default 0.5). Description: "Check whether documentation covering a topic already exists. Returns existence boolean, confidence score, and matching documents. Use before creating new docs to avoid duplication."

    6. **get_diagrams** (MCPR-07): Adapt SQL from server.mjs /diagrams endpoint. Include computed fields: active_url (curated_url || figjam_url || null), stale boolean. Input: repo(string?), stale_only(boolean, default false), limit(int 1-100, default 50). Description: "Get diagram registry. Filter by repository and/or stale status. Returns diagram names, URLs, and staleness information."

    **Transport connection:**
    ```javascript
    const transport = new StdioServerTransport();
    ```
