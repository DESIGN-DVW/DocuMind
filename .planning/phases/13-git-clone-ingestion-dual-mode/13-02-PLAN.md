<task type="auto">
  <name>Task 2: Restructure docker-compose.yml for dual mode and update .env.example</name>
  <files>docker-compose.yml, .env.example</files>
  <action>
1. Replace the entire `docker-compose.yml` with a dual-mode configuration:

```yaml
services:
  documind:
    build: .
    ports:
      - "${DOCUMIND_PORT:-9000}:9000"
    volumes:
      - documind_data:/app/data
      # --- Mount mode (REPO_MODE=mount, default) ---
      # Bind-mount host repos read-only. Adjust left side to your repos path.
      - ${REPOS_HOST_PATH:-/Users/Shared/htdocs/github/DVWDesign}:/repos:ro
      # --- Clone mode (REPO_MODE=clone) ---
      # Named volume persists cloned repos across container restarts.
      # Uncomment the line below and comment out the bind mount above.
      # - documind_repos:/app/repos
    environment:
      # --- Mode selection ---
      REPO_MODE: "${REPO_MODE:-mount}"
      # --- Mount mode ---
      DOCUMIND_REPOS_DIR: "${DOCUMIND_REPOS_DIR:-/repos}"
      CHOKIDAR_USEPOLLING: "true"
      CHOKIDAR_INTERVAL: "2000"
      # --- Clone mode (set these when REPO_MODE=clone) ---
      # DOCUMIND_REPOS: "${DOCUMIND_REPOS}"
      # GIT_TOKEN: "${GIT_TOKEN}"
    restart: unless-stopped

volumes:
  documind_data:
  # Uncomment for clone mode to persist cloned repos
  # documind_repos:
```

Key design decisions:
- `REPO_MODE` defaults to `mount` via `${REPO_MODE:-mount}` — existing behavior preserved
- Mount mode bind mount uses `${REPOS_HOST_PATH:-fallback}` so the host path is configurable without editing the file
- Clone mode vars (`DOCUMIND_REPOS`, `GIT_TOKEN`) are commented out by default — user uncomments when switching
- `GIT_TOKEN: "${GIT_TOKEN}"` reads from host env at `docker compose up` time — NEVER baked into image
- `DOCUMIND_REPOS_DIR` is set to `/repos` for mount mode; for clone mode the user changes it to `/app/repos`
- `documind_repos` named volume is commented out; user uncomments for clone mode

2. Update `.env.example` — read the current file first, then append the new variables at the end (or in the appropriate section). Add these entries:

```bash
# --- Ingestion Mode ---
# mount = scan bind-mounted host directories (default)
# clone = clone repos from GitHub on startup, pull on schedule
REPO_MODE=mount

# --- Clone Mode Settings (only used when REPO_MODE=clone) ---
# Comma-separated org/repo slugs to clone
# DOCUMIND_REPOS=DVWDesign/DocuMind,DVWDesign/RootDispatcher
# GitHub personal access token (never commit this!)
# GIT_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
# When using clone mode, set DOCUMIND_REPOS_DIR=/app/repos
# DOCUMIND_REPOS_DIR=/app/repos

# --- Mount Mode Settings (only used when REPO_MODE=mount) ---
# Host path for bind mount (used in docker-compose.yml)
# REPOS_HOST_PATH=/Users/Shared/htdocs/github/DVWDesign
```

Ensure the clone mode vars are commented out (prefixed with `#`) since they are optional and potentially sensitive.
