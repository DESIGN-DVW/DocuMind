  <name>Task 1: Create ingestion module and add REPO_MODE to env.mjs</name>
  <files>config/env.mjs, daemon/ingestion.mjs</files>
  <action>

1. In `config/env.mjs`, add a `REPO_MODE` export after the `REPOS_LIST` export block:

```javascript

export const REPO_MODE = process.env.REPO_MODE ?? 'mount';

```

Add it inside the "REPOSITORY DISCOVERY" section. No validation here — env.mjs stays side-effect-free.

2. Create `daemon/ingestion.mjs` with two exported async functions:

**`initIngestion()`** — called once at startup:

- Import `REPO_MODE`, `REPOS_LIST` from `config/env.mjs`

- If `REPO_MODE === 'mount'`: log `[ingestion] mount mode — skipping clone` and return immediately

- If `REPO_MODE !== 'clone'`: log error and `process.exit(1)`

- If `REPOS_LIST` is null or empty: log error "REPO_MODE=clone but DOCUMIND_REPOS is not set" and `process.exit(1)`

- Read `GIT_TOKEN` from `process.env.GIT_TOKEN` (not from env.mjs — secrets should not be module-level exports)

- If no `GIT_TOKEN`: log error and `process.exit(1)`

- Set `CLONE_DIR` to `/app/repos` (constant at module level)

- Create `CLONE_DIR` with `fs.mkdir(CLONE_DIR, { recursive: true })`

- For each slug in `REPOS_LIST` (format: `org/repo`):

  - Derive repo name: `slug.split('/').pop()`

  - Check if `path.join(CLONE_DIR, name, '.git')` exists via `fs.access()`

  - If exists: log skip and continue

  - If not: clone with `execFileAsync('git', ['clone', '--depth=1', url, repoPath])` where `url = \`https://${token}@github.com/${slug}.git\``

- After all clones complete, log ready message

**CRITICAL:** `initIngestion()` does NOT mutate `process.env.DOCUMIND_REPOS_DIR`. The docker-compose.yml for clone mode sets `DOCUMIND_REPOS_DIR=/app/repos` as an env var, so `REPOS_DIR` in env.mjs is already correct at import time.

**`pullAllRepos()`** — called by scheduler cron:

- If `REPO_MODE !== 'clone'`: return empty array

- Read `CLONE_DIR` directory entries

- For each directory entry:

  - Run `execFileAsync('git', ['-C', repoPath, 'pull', '--ff-only'])`

  - If stdout does NOT contain "Already up to date", add repo name to `updated` array

  - On error: log warning, attempt fallback: `git fetch origin` then `git reset --hard origin/HEAD`, add to `updated` array. If fallback also fails, log error and skip.

- Return `updated` array (repo names that changed)

Use `child_process.execFile` (via `promisify`) — NOT `exec()` — to avoid shell injection. Import from `node:child_process` and `node:util`.

   - After the existing `CRON_RELINK` cron registration (before the console.log summary), add a conditional block:

```javascript

// Clone mode: pull repos and re-scan changed ones on hourly schedule
if (REPO_MODE === 'clone') {
  cron.schedule(CRON_HOURLY, async () => {
    console.log('[scheduler] Pulling cloned repos...');
    try {
      const updated = await pullAllRepos();
      if (updated.length === 0) {
        console.log('[scheduler] All repos up to date');
        return;
      }
      for (const name of updated) {
        console.log(`[scheduler] Re-scanning ${name} after pull...`);
        await runScan(db, ctx, { mode: 'incremental', repo: name });
      }
      console.log(`[scheduler] Pull + re-scan complete: ${updated.length} repo(s) updated`);
    } catch (err) {
      console.error('[scheduler] Pull cron failed:', err.message);
    }
  });
  console.log(`  ${CRON_HOURLY}     — clone mode: git pull + re-scan`);
}

```

Note: This registers a SECOND cron on the same `CRON_HOURLY` schedule. That is intentional — the existing hourly cron does a general incremental scan; this one does git pull first for clone mode. node-cron handles multiple jobs on the same schedule correctly (they run sequentially per tick). However, to avoid scanning twice, consider whether the pull cron should replace the existing hourly cron when in clone mode, or complement it. The simplest approach: let both run — the incremental scan after pull handles the freshly pulled files, and the general incremental scan picks up any other changes. The scan is idempotent (content_hash dedup), so double-scanning is harmless but slightly wasteful. Keep it simple for now.
