The module exports two functions. Both accept `(db, kuzuDb)` where `db` is a better-sqlite3 Database instance and `kuzuDb` is the kuzu.Database singleton owned by server.mjs.

**REL_TYPES constant** (module-level, matches frozen kuzu-init.mjs schema exactly):
```javascript
const REL_TYPES = [
  'imports', 'dispatched_to', 'supersedes', 'related_to',
  'parent_of', 'variant_of', 'depends_on', 'generated_from'
];
```

**insertEdge(conn, edge) — private async function:**
- Dispatches on `edge.relationship_type` to build a typed Cypher CREATE
- Parses `edge.metadata` as JSON (default `{}` if null/empty)
- All edge CREATEs use `MATCH (src:Document {id: $src}), (tgt:Document {id: $tgt})` pattern first, then CREATE the edge
- Property mapping per type (from frozen schema):
  - `imports`: `{weight: $w, link_text: $lt}` — `w = edge.weight ?? 1.0`, `lt = meta.link_text ?? ''`
  - `dispatched_to`: `{target_repo: $repo}` — `repo = meta.target_repo ?? ''`
  - `supersedes`: `{confidence: $conf}` — `conf = meta.confidence ?? edge.weight ?? 0.7`
  - `related_to`: `{weight: $w, reason: $reason}` — `w = edge.weight ?? 0.3`, `reason = meta.reason ?? ''`
  - `variant_of`: `{similarity_score: $score}` — `score = meta.similarity_score ?? edge.weight ?? 0.0`
  - `parent_of`, `depends_on`, `generated_from`: no properties — use `CREATE (src)-[:${type}]->(tgt)`
- Pass parameters as the second argument to `conn.query(cypher, params)` — kuzu@0.11.3 supports named params with `$` prefix

**syncToKuzu(db, kuzuDb) — exported async function:**
Steps in order:
1. Open `const conn = new kuzu.Connection(kuzuDb)` in a try/finally
2. SELECT all documents: `SELECT id, path, repository, filename, category FROM documents`
3. For each doc, MERGE the Document node:
   ```
   MERGE (d:Document {id: $id})
   SET d.path = $path, d.repository = $repo, d.filename = $filename, d.category = $cat
   ```
   with params `{ id: doc.id, path: doc.path ?? '', repo: doc.repository ?? '', filename: doc.filename ?? '', cat: doc.category ?? '' }`
   Note: `id` from better-sqlite3 is already a JS integer — pass directly, do NOT coerce to string or BigInt
4. Drop all existing edges across all 8 tables (makes re-sync idempotent):
   ```javascript
   for (const rel of REL_TYPES) {
     await conn.query(`MATCH ()-[r:${rel}]->() DELETE r`);
   }
   ```
5. SELECT all edges: `SELECT source_doc_id, target_doc_id, relationship_type, weight, metadata FROM doc_relationships`
6. For each edge, call `await insertEdge(conn, edge)`
7. In finally: `try { conn.close(); } catch (_) {}`
8. Return `{ nodeCount: docs.length, edgeCount: edges.length }`
9. Log: `[kuzu-sync] Sync complete: ${nodeCount} nodes, ${edgeCount} edges mirrored`
10. Do NOT call `process.exit()` — this is a daemon-context module

**rebuildKuzuGraph(db, kuzuDb) — exported async function:**
Full wipe (all edges first, then all nodes) then re-populate:
1. Open `const conn = new kuzu.Connection(kuzuDb)` in a try/finally
2. Delete all edges: `for (const rel of REL_TYPES) await conn.query('MATCH ()-[r:${rel}]->() DELETE r')`
3. Delete all nodes: `await conn.query('MATCH (d:Document) DELETE d')`
4. Close conn in finally
5. Then call `return syncToKuzu(db, kuzuDb)` — reuse sync for the re-population
6. Log before step 1: `[kuzu-sync] Full rebuild: dropping all Kuzu nodes and edges...`

**Important constraints:**
- Never open a second `kuzu.Database` — only use the `kuzuDb` parameter
- Never call `process.exit()` — standalone scripts handle that
- Connection lifecycle: always open a new Connection per function call, close in finally
- The module uses `import kuzu from 'kuzu'` (ESM default import confirmed in Phase 16-01)
