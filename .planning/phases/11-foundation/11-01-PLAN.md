  <name>Task 1: Create config/env.mjs and .env.example</name>
  <files>config/env.mjs, .env.example, .gitignore</files>
  <action>
Create `config/env.mjs` as the single source of truth for all runtime configuration. This module:

1. Computes ROOT as `path.resolve(__dirname, '..')` at the top
2. Calls `process.loadEnvFile(path.join(ROOT, '.env'))` inside a try/catch (silent skip if .env missing — Docker passes vars directly)
3. Exports named constants with `??` defaults:
   - `ROOT` — project root directory
   - `PORT` — `Number(process.env.PORT ?? 9000)`
   - `DB_PATH` — `path.resolve(ROOT, process.env.DOCUMIND_DB ?? 'data/documind.db')`
   - `PROFILE_PATH` — `path.resolve(ROOT, process.env.DOCUMIND_PROFILE ?? 'config/profiles/dvwdesign.json')`
   - `REPOS_DIR` — `process.env.DOCUMIND_REPOS_DIR ? path.resolve(process.env.DOCUMIND_REPOS_DIR) : null` (null means use profile/registry)
   - `REPOS_LIST` — `process.env.DOCUMIND_REPOS ? process.env.DOCUMIND_REPOS.split(',').map(s => s.trim()).filter(Boolean) : null`
   - `CRON_HEARTBEAT` — `process.env.DOCUMIND_CRON_HEARTBEAT ?? '*/15 * * * *'`
   - `CRON_HOURLY` — `process.env.DOCUMIND_CRON_HOURLY ?? '0 * * * *'`
   - `CRON_DAILY` — `process.env.DOCUMIND_CRON_DAILY ?? '0 2 * * *'`
   - `CRON_WEEKLY` — `process.env.DOCUMIND_CRON_WEEKLY ?? '0 3 * * 0'`
   - `CRON_RELINK` — `process.env.DOCUMIND_CRON_RELINK ?? '0 */6 * * *'`

IMPORTANT: config/env.mjs must NOT import from config/constants.mjs (circular dependency pitfall from RESEARCH.md). It only reads process.env and uses Node built-ins.

Create `.env.example` with all documented variables, grouped by section (Server, Database, Profile, Repository Root, Explicit Repo List, Cron Schedules). Include comments explaining each var. Set DOCUMIND_REPOS_DIR default to `/Users/Shared/htdocs/github/DVWDesign`. Show Docker example as a comment. Cron vars should be commented out (defaults are fine for most users).

Add `.env` to `.gitignore` (currently missing — Pitfall 5 from RESEARCH.md). Add it after the existing "Database files" section.
