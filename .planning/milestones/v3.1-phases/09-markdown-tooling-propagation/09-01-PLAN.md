Create `scripts/propagate-lint-rules.mjs` that propagates DVW001 + MD060A to all target repos.

The script must:

1. Define the target repo list (all DVWDesign repos with markdown + package.json, excluding DocuMind):

   - @figma-agents, @figma-core, Aprimo, CampaignManager, Contentful, Figma-Plug-ins, FigmaDSController, GlossiaApp, LibraryAssetManager, RandD, RootDispatcher, any2figma, mjml-dev-mode-proposal, mjml-dev-mode, mjml_mcp, shared-packages

2. The DVWDesign base path is `/Users/Shared/htdocs/github/DVWDesign/`.

3. For each target repo:

   a. Create `config/rules/` directory if it does not exist
   b. Copy `table-separator-spacing.cjs` from DocuMind's `config/rules/` into the target repo's `config/rules/`
   c. Copy `force-align-table-columns.cjs` from DocuMind's `config/rules/` into the target repo's `config/rules/`
   d. Create `.markdownlint-cli2.jsonc` in repo root with this content:

      ```jsonc

      // markdownlint-cli2 configuration
      // Custom rules propagated from DocuMind
      {
        "customRules": [
          "./config/rules/table-separator-spacing.cjs",
          "./config/rules/force-align-table-columns.cjs"
        ],
        "ignores": [
          "node_modules",
          ".git",
          "dist",
          "build",
          ".planning"
        ]
      }

      ```

      NOTE: Do NOT include `"config": { "extends": ... }` — only DocuMind has the full .markdownlint.json config. Target repos get only the custom rules. If the target already has a `.markdownlint-cli2.jsonc`, merge the customRules array into the existing config rather than overwriting.
   e. Check if `markdownlint-cli2` is in devDependencies in package.json. If not, add it. Same for `markdownlint-rule-force-align-table-columns`. Use `npm install --save-dev` (or detect pnpm from lockfile and use `pnpm add -D`).

4. Print a summary table at the end showing each repo and what was done (copied rules, created config, installed deps).

5. Accept a `--dry-run` flag that prints what would be done without making changes.

6. Accept a `--repo <name>` flag to target a single repo instead of all.

Use `node:fs`, `node:path`, `node:child_process` (execSync). Use chalk for colored output. Import from DocuMind's own node_modules since the script runs from DocuMind.
