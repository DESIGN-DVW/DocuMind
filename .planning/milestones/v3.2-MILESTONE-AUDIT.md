### Partial Wiring

**docker-compose.yml clone-mode DOCUMIND_REPOS_DIR** — The commented clone-mode block in docker-compose.yml does not include `DOCUMIND_REPOS_DIR=/app/repos`. An operator who enables clone mode without reading DOCKER-USAGE.md docs would leave `DOCUMIND_REPOS_DIR=/repos` (the mount-mode default), causing `discoverRepos()` to scan the empty bind-mount path instead of `/app/repos` where repos were cloned. The code is correct; the compose file documentation is incomplete.

**Fix:** Add `# DOCUMIND_REPOS_DIR: "/app/repos"` to the clone-mode comment block.
