## Integration Gaps

Three parameter/field name mismatches between Phase 8 slash commands (RootDispatcher) and MCP tool schemas (DocuMind):

### 1. /figma-diagram → register_diagram (SLSH-02)

- **Tool schema:** `mermaid_path: z.string()`
- **Slash command:** passes `mmd_path`
- **Impact:** Tool receives undefined path, returns file-not-found error
- **Fix:** Update `figma-diagram.md` to pass `mermaid_path` instead of `mmd_path`

### 2. /figma-curate → curate_diagram (SLSH-03)

- **Tool schema:** `name: z.string()`
- **Slash command:** passes `diagram_name`
- **Impact:** `relinkDiagram` receives undefined, returns "Diagram not found"
- **Fix:** Update `figma-curate.md` to pass `name` instead of `diagram_name`

### 3. /diagram-registry → get_diagrams (SLSH-01)

- **Tool schema returns:** `mermaid_path`, `figjam_url`, `stale`/`is_stale`, `active_url`
- **Slash command expects:** `mmd_path`, `png_path`, `generated_url`, `status`
- **Impact:** Agent filtering logic produces zero matches (fields don't exist)
- **Fix:** Update `diagram-registry.md` response documentation to use actual field names

### Root Cause

Phase 8 planner used assumed parameter names instead of reading the actual Zod schemas from `daemon/mcp-server.mjs`. The Phase 8 verifier checked that commands referenced the correct tool names but did not validate parameter-level compatibility.
